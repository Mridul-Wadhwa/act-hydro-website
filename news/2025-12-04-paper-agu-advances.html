New paper in AGU Advances

Toward Co‑Designed Earth System Models: Reflecting End‑User Priorities in Local Applications From a Modeler's Perspective by Yifan Cheng et al. is now published in AGU Advances.

This paper discusses why and how Earth System Models (ESMs) can be co‑designed with end users to improve local relevance and usability. The authors present two linked case studies: the Arctic Rivers Project (high‑resolution climate and hydrology for Alaska) and the Mid‑Klamath Project (collaboration with the Karuk Tribe on prescribed burning and hydrology).

As shown in Figure 2, the team used early engagement (advisory councils and workshops), iterative demonstrations, and ongoing feedback loops to surface local priorities — for example, monitoring practices and resource management concerns — that directly shaped data collection and model choices. This practical, iterative approach made technical outputs more relevant to local decision makers.

Building on these case studies, the paper proposes a practical four‑level framework for co‑design:

  1. Co‑design Configuration — set goals, partners, and processes;
  2. Model Tuning — adjust model parameters and evaluation to reflect local metrics;
  3. Incorporate Contextual Knowledge — integrate monitoring protocols and local observations;
  4. Co‑develop New Model Functions — add new model capabilities driven by partner needs.

Figure 3 visualizes this roadmap: teams can start with configuration and progressively deepen engagement and technical co‑development. The framework is useful because it gives model teams clear entry points for collaboration and a path to scale up co‑design efforts as trust and capacity grow.
